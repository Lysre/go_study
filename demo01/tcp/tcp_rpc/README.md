# tcp组包


~~~go
通信使用tcp/ip网络协议，socket同步短连接。
报文使用固定报文头+xml报文格式，编码使用GBK编码。
报文包括 4位报文长度+报文头+xml报文内容+128位 的报文签名。
其中4位长度是报文头+xml报文的长度+128位签名字段的长度（不包括自己的4个字节）。
请求报文发送后等待应答报文的返回，超时时间定义是60秒。
报文头定义：
序号	字节长度	字段说明	补充规则
1	4	版本号	固定 0000，不足补空格
2	15	机构编号	与 XML 内mer_code一致，补空格
3	10	交易码	后补空格
4	16	流水号	格式YYYYMMDD12345678，补空格
5	1	报文类型	请求q / 应答s
6	2	错误码	请求填空格；应答：00/BK/99/xx
7	100	错误信息	请求填空格；应答失败填内容，补空格

如果请求报文有错误无法处理，那么直接返回没有xml报文的报文头。
报文头使用空格填充没有使用的字符。
128位签名信息使用MD5处理（计算出来的MD5值需要展开成16进制可见字符）报文头和xml报文(注意，报文头不包括最前面的4个长度字符)，然后使用dsa算法进行签名，签名后的数据需要展开成16进制的可见字符，展开后的数据长度是92到96位之间。
签名字段的组成是 3位长度+签名信息+补足空格 ，例如：12345678….n，一共92位长度，那么签名字段就是09212345678….n            ….。
一共 3位长度+92位数据+33个空格。
在交易报文中，请求xml报文的所有域大部分都是必须的域，返回的xml域在交易成功的情况下都是必须的，如果返回交易失败，那么xml中的报文域就不一定都会有。
极端情况下，xml报文域都不会有，只有一个拒绝报文头。
